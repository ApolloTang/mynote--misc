::: install and update package.json file:

    --save

        this option will add the install module(s) to
        dependencies item to package.json

            "dependencies": {
                "grunt": "0.4.2"
            }

    --save-dev

        this option will add the install module(s) to
        devDpendencies item to package.json

          "devDependencies": {
            "grunt": "^0.4.5",
            "grunt-contrib-uglify": "^0.4.0"
          }

::: [!] About share folder in virture box

        need --no-bin-links for plugin installation whenever you
        install something inside of a vbox shared folder because
        it's not possible to store symlinks in those places.

        eg:  sudo npm install --no-bin-links grunt grunt-contrib-uglify

::: To specify a target directory when running npm install <package>:

    use the --prefix option:

        mkdir -p ./install/here/node_modules
        npm install --prefix ./install/here <package>

    The package(s) will then be installed in

         ./install/here/node_modules.

    [!] The mkdir is needed since npm might otherwise choose
    an already existing node_modules directory higher up in
    the hierarchy.

    -p in mkdir will also create all directories leading up
    to the given directory that do not exist already.

        See npm documentation on folders.
            https://www.npmjs.org/doc/files/npm-folders.html#More-Information

    source:
    http://stackoverflow.com/questions/14469515/how-to-npm-install-to-a-specified-directory
